I use Obsidian for managing my personal tasks.
Simple tasks are managed via checkboxes in daily notes and/or Tasks' plugin.
More complex tasks - "Projects" as in David Allen's GTD, consisting of several tasks - are managed like this:
1. one note for one project
2. projects statuses are via "status" property (todo, in-progress, blocked, backlog, done, cancelled)
3. child notes are created as needed, just linked from project note. usually meeting or research notes
4. tasks checklist in each project, but often non-connected tasks from daily notes care created ad hoc
5. kanban bases view based on status for seeing projects at a glance

current challenges:
1. I want to see a neat list of active ("current") projects in a side pane. I have bases list view for this but it's not optimal
2. I want to easily set, change, and see perceived "progress" for project; counting based on tasks will not do
3. I want to see if project is delegated
4. I want to see if project has a due date

I am thinking about creating a simple Obsidian plugin that will add improved list, table, and kanban views for my workflow.
